# 代码规范

- Python 官方提供有一系列 PEP（Python Enhancement Proposals） 文档
- 其中第 8 篇文档专门针对 Python 的代码格式 给出了建议，也就是俗称的 PEP8
- 文档地址：https://www.python.org/dev/peps/pep-0008/
- 谷歌有对应的中文文档：http://zh-google-styleguide.readthedocs.io/en/latest/google-python-styleguide/python_style_rules/

## 代码风格指南

- 缩进：使用 4 个空格的缩进，不要使用 tab 键
- 行宽：每行代码不超过 79 个字符，超过 79 个字符需要换行
- 空行：空行用来分隔函数和类，以及逻辑代码块
- 空格：在二元运算符、逗号、分号、冒号、括号等符号前后加空格
- 注释：注释应该是完整的句子，以句号结尾
- 文档字符串：应该使用三引号，并且应该在函数定义的上面，描述函数的功能和用法
- 变量名：使用小写字母和下划线，不要使用驼峰命名法
- 类名：使用驼峰命名法
- 函数名：使用小写字母和下划线，不要使用驼峰命名法
- 常量名：全部大写，单词之间用下划线连接
- 异常名：使用驼峰命名法，以 Error 结尾
- 包名：使用小写字母和下划线，不要使用驼峰命名法
- 模块名：使用小写字母和下划线，不要使用驼峰命名法
- 类型名：使用驼峰命名法
- 实例名：使用驼峰命名法
- 变量名：使用小写字母和下划线，不要使用驼峰命名法
- 常量名：全部大写，单词之间用下划线连接
- 布尔值：使用 is True 和 is False，不要使用 == True 和 == False
- 条件表达式：使用括号，并且不要使用行连接符
- 循环：使用 while 循环，而不是 for 循环
- 列表推导式：使用括号，并且不要使用行连接符
- 字典推导式：使用括号，并且不要使用行连接符
- 字符串格式化：使用 f-string，而不是 % 格式化
- 异常处理：使用 try-except 语句，并且捕获具体的异常类型
- 文档字符串：应该使用三引号，并且应该在函数定义的上面，描述函数的功能和用法
